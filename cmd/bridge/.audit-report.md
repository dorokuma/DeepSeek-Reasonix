## 1. 对话 draft → 全文正式消息（过长拆段），无悬空 draft ✅ 已实现

- `splitTelegramText()`（telegram_text.go:59-99）按 4096 runes 分片，优先换行符断开。`closeDraft()`（handler.go:347-377）编辑草稿第一条消息为首段，余段 `sendMessage` 续发。
- 空内容走 `dismissDraft` 删除草稿；`TurnStarted` 自动 dismiss 上轮草稿（sink.go:55-58）。

## 2. Message 不翻倍 ✅ 已实现

- 草稿首次 `sendMessage`，后续流式更新用 `editMessageText`（handler.go:315-343）。定稿编辑已有消息不新建，首段编辑、续段新发，不重复。

## 3. 审批/Ask 不堵 Sink ✅ 已实现

- `event.ApprovalRequest`（sink.go:179-183）和 `event.AskRequest`（sink.go:185-189）emit 后通过回调立即发送 Telegram 消息，不等待用户。注释标注「§4.9: non-blocking — emit immediately, return」。

## 4. 永久允许后重启仍免问（WORK_DIR 配置可写）🔶 部分实现

- 键盘「📌 持久允许」按钮存在（approve.go:52），调用 `ctrl.Approve(..., true, false, true)` 由 reasonix 核心写 `reasonix.toml`。
- `validatePaths()`（main.go:112-114）检查 `WORK_DIR/reasonix.toml` 父目录可写。
- **缺口**：`stateStore`（persist.go）定义了 chat→path 索引持久化但**从未被实例化**。`SessionManager.index` 纯内存，重启后连接丢失。

## 5. /stop 取消 + dismiss ✅ 已实现

- `handler.go:126-128`：`/stop` → `b.sm.Stop(chatID)` → `ctrl.Cancel()`，发送「⏹ 已停止」。
- 草稿在下一轮 `TurnStarted` 自动 dismiss，无残留风险。

## 6. /new：Wait 后索引正确；首条消息无索引时能 mint path 🔶 部分实现

- **首条 mint path**：`ensureController` → `resolvePath()`（session.go:56-67）在 index 为空时自动 `agent.NewSessionPath()` 生成新路径——正确。
- **/new 流程不完整**：当前仅 `b.sm.Stop(chatID)` + 返回提示（handler.go:164-166），未按设计执行 `Submit("/new") → Wait() → 索引更新`。无 Submit、无 Wait、无索引更新。
- 同 #4，索引不持久化。

## 7. /model 列表能显示；切换失败不丢会话；成功旧 Close 🔶 部分实现

- **切换**：完全未实现。无 Snapshot → Build(新模型) → Load+Resume → 原子替换 → 旧 Close 流程。

## 8. /status API 签名正确 🔶 部分实现

- 当前仅展示 `ctrl.Running()`、`ctrl.Turn()`、`ctrl.Label()`（handler.go:141-151）。
- 设计要求的缺失字段：`LastUsage()`、`SessionCache()`、`SessionCost()`、`Balance()`、`MultiAgentControl().List()`、`SessionPath()`。Sink 虽记录了 `lastUsage`/`sessionCost`/`sessionCache` 但未暴露。

## 9. `!`/`/compact` 无 TurnStarted 不误关流；`/memory`/`/rewind` 无 Done 不卡死 🔶 部分实现

- **无 TurnStarted 保护**：sink.go 各事件前 `if !s.dialogueUI { return }`，无 TurnStarted 时事件静默丢弃，不误操作。
- **路由拦截**：`/compact`、`/memory`、`/rewind` 不在 `handleCommand` switch 中，走 `default` 返回「未知命令」，根本到不了 sink。与 #10 同源。
- **无 Done 卡死风险**：若通过 Submit 传入只读命令但不产生 TurnDone，`finalizeTurn()` 永不调用，草稿挂起。当前无处理机制。

## 10. 手打 /memory 走 Submit 生效 ❌ 未实现

- `/memory` 落入 `handleCommand` default 分支返回「未知命令」（handler.go:183-185），不进入 Submit 流程。原因同 #9：所有 `/` 开头消息先路由到 `handleCommand`，而该函数只识别有限白名单。

## 11. 加密 Resume；旧会话可续；损坏换 path ✅ 已实现

- `sessioncrypt.go`：完整 AES-256-GCM 加密/解密，含魔法头检测、双加密保护、透明读写、密钥轮换。
- `persist.go`：`migrateOldSessions()` 转旧 `.jsonl` 为 `.jsonl.enc`；`saveAll`/`upsert` 原子写入。
- `ensureController`（session.go:95-107）：`LoadSession` 失败（损坏）→ 记录日志 → 新路径 → 设置。成功则 `ctrl.Resume(sess, path)`。
- **缺口**：chat→path 索引不持久化，重启后即使 session 文件存在也无法自动恢复。

## 12. 空闲 Build+Resume ✅ 已实现
