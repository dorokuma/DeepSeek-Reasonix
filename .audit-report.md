## 总体评价

Reasonix 在安全方面表现出**较高的工程素养**：session 有 AES-256-GCM 加密、web_fetch 有 SSRF guard、子进程有凭据剥离、HTTP API 有 CSRF 防护、权限系统设计精良。但审计仍发现了 **2 个 P0、11 个 P1、12 个 P2** 级别问题。

## 正面发现（安全亮点）

1. **Session 加密** — AES-256-GCM + O_EXCL 防 TOCTOU + 0o600 权限
2. **web_fetch SSRF guard** — dial-time IP 检查 + DNS rebinding 防护 + CGNAT 覆盖
3. **凭据剥离** — 环境变量中的 API key 在子进程中被系统性清除（30+ 后缀模式）
4. **权限系统** — 纯规则评估，deny > ask > allow 优先级，shell 语法检测
5. **CSRF 防护** — Content-Type 强制 + Origin/Referer 验证 + 1 MiB body 限制
6. **路径遍历防护** — filepath.Rel 验证、system_prompt_file 拒绝绝对路径和 `..`
7. **Hook 信任模型** — 项目 hook 需显式信任，输出限制 256 KB
8. **CI/CD 权限最小化** — GitHub Actions 使用 `contents: read` 最小权限
9. **配置文件 gitignore** — .env、reasonix.toml 等本地配置文件均已 gitignore
